## 추가된 내용

### 4.5 공급처 관리 (Supplier)
- 공급처 코드, 공급처명, 담당자, 연락처
- 거래 조건 (결제일, MOQ)
- Supplier 데이터 모델

### 4.6 발주 관리 (Ordering)
- **발주점 (Reorder Point)**: 이 수량 이하 시 알림
- **발주 수량 (Reorder Qty)**: 기본 발주 수량
- **리드타임 (Lead Time)**: 발주~입고 소요일
- **안전 재고**: 최소 유지 재고 수량
- **발주 흐름**: 감지 → 알림 → 발주서 → 전송 → 입고

### 데이터 모델
- Supplier: 공급처 정보
- PurchaseOrder: 발주서

### API
- /api/suppliers: 공급처 CRUD
- /api/purchase-orders: 발주서 CRUD
- /api/reorder-alerts: 발주 필요 상품 목록
